Knife4j

  # 支持 swagger3
spring:
  mvc:
    pathmatch:
      matching-strategy: ant_path_matcher

Knife4jConfig.java

@Configuration
@EnableSwagger2
@Profile({"dev", "test"})
public class Knife4jConfig {
    @Bean
    public Docket defaultApi2() {
        return new Docket(DocumentationType.SWAGGER_2)
                .apiInfo(new ApiInfoBuilder()
                        .title("接口文档")
                        .description("Book Management System")
                        .version("1.0")
                        .build())
                .select()
                // 指定 Controller 扫描包路径
                .apis(RequestHandlerSelectors.basePackage("com.puff.bkms.controller"))
                .paths(PathSelectors.any())
                .build();
    }
}

知识点

  1. Spring security+jwt 对用户进行认证时，既然已经将用户信息保存到SecurityContextHolder中了，那么为什么还需要返回token

    将用户信息保存到SecurityContextHolder中是为了在服务器端进行身份验证和授权，以便在处理请求时能够获取到用户的身份和权限信息

    返回 token 主要是为了在客户端和服务器之间支持无状态的认证和授权机制，从而提高系统的可伸缩性和安全性。通过 token，客户端可以在每次请求中携带身份验证信息，而不需要依赖会话状态。（浏览器关闭时服务器通常会清除session，但是Token是保存在浏览器的本地存储或内存中的）

    因此，返回 token 和将用户信息保存到 SecurityContextHolder 中是两个并行的操作，它们分别用于支持客户端和服务器端的身份验证和授权过程。

    使用 token 进行身份验证和授权会增加客户端访问服务器的成本，因为每次请求都需要携带 token。这可能会导致额外的网络开销和资源消耗。然而，使用 token 进行认证和授权也带来了一些优势，例如支持无状态的通信、提高了安全性和可伸缩性等。因此，在设计系统时，需要权衡这些因素，并根据具体的需求和场景来选择合适的认证和授权机制。

  2. Token的作用

    主要是安全性和可伸缩性

    安全性体现在使用 token 进行身份验证和授权时，因为 token 可以包含加密的信息，确保了通信的安全性。此外，使用 token 进行认证和授权可以减少对会话状态的依赖，降低了 CSRF（跨站请求伪造）和 XSS（跨站脚本攻击）等安全风险。

    可伸缩性体现在使用 token 进行认证和授权时，因为有状态保存用户状态，不支持集群化部署。而无状态的通信使得系统可以更容易地进行水平扩展，而不需要维护会话状态。这意味着系统可以更好地应对高负载和大规模的用户请求，提高了系统的可伸缩性和性能表现。

2. 资源过滤问题

class path resource [scripts/counter.lua] cannot be opened because it does not exist

        <resources>
            <resource>
                <directory>src/main/java</directory>
                <includes>
                    <include>**/*.xml</include>
                </includes>
                <filtering>false</filtering>
            </resource>
            <resource>
                <directory>src/main/resources</directory>
                <includes>
                    <include>**/*.yml</include>
                    <include>**/*.xml</include>
                    <include>**/*.lua</include>
                </includes>
                <filtering>false</filtering>
            </resource>
        </resources>

添加<include>**/*.lua</include>后还需要重新载入一下pom.xml，不然修改的内容不会生效

其实出现not exist的错误直接把文件复制到target/classes/**下面再重新启动测试一下即可
