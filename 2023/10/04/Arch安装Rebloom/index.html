Arch安装Rebloom

2023/10/4 linux Redis

曲折的Rebloom安装之路

  • 前往github获取源码(https://github.com/RedisBloom/RedisBloom)
git clone --recursive https://github.com/RedisBloom/RedisBloom.git
cd RedisBloom
./sbin/setup
bash -l
make

1.启动文件setup执行报错

ERROR: Running makepkg as root is not allowed as it can cause permanent, catastrophic damage to your system.

参考链接：

pip(3) install，完美解决 externally-managed-environment

选择解决方式：

  • 方案一、(粗暴) 去掉这个提示

强制删除此警告，回归到熟悉的操作。

将 “x” 替换为实际版本。

1 sudo mv /usr/lib/python3.x/EXTERNALLY-MANAGED /usr/lib/python3.x/EXTERNALLY-MANAGED.bk

和之前一样，现在您可以直接运行 pip(3) install package_name 命令来安装python模块。

2.执行启动文件setup报错

FileNotFoundError: Failed to find yay or trizen, for aur package installation.

  • 尝试1：

    下载terizen

    出现新的报错

    ERROR: Running makepkg as root is not allowed as it can cause permanent,
catastrophic damage to your system.

    tarizen无法使用管理员权限进行下载软件，于是找到system-setup.py文件

        def common_last(self):
        if self.dist == "arch":
            # self.install("lcov-git", aur=True)
            eprint("yes")
        else:

    将其中的self.install("lcov-git", aur=True)注释掉，然后手动执行trizen --noconfirm -S lcov-git

    之后再继续执行启动文件./sbin/setup即可

安装RedisBloom模块最好是先配置redis.conf文件，然后动态加载RedisBoom模块。因为只动态加载，在Redis重启后就失效了，但是只在redis.conf中配置，又没办法动态生效，必须重启。

安装成功后通过module list进行查看是否配置成功

127.0.0.1:6379> module list
1) 1) "name"
   2) "bf"
   3) "ver"
   4) (integer) 999999
   5) "path"
   6) "/etc/redis/RedisBloom/bin/linux-x64-release/redisbloom.so"
   7) "args"
   8) (empty array)

至此，RedisBloom模块安装完成
