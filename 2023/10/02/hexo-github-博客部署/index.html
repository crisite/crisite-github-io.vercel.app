hexo-github 博客部署

2023/10/2

部署基于 Hexo-github 的个人博客

部署

环境

Nodejs

Git

Hexo

步骤

  1. 安装 Hexo

npm install -g hexo-cli

安装完成后输入hexo version

创建新的文件夹crisite.github.io

hexo init
hexo start

启动 hexo 服务之后进入浏览器即可访问本地博客

常用命令

hexo new “postName” #新建文章
hexo new page “pageName” # 新建页面
hexo generate # 生成静态页面至 public 目录
hexo server # 启动服务器(默认端口 4000，’ctrl+c’关闭 server)
hexo deploy # 项目部署
hexo help # 查看帮助
hexo version # 查看 Hexo 的版本
hexo clean # 清除 Hexo 的缓存

注：在使用部署命令时，需要先用 npm 安装 hexo-deployer-git 插件：

npm install hexo-deployer-git --save

  1. Github 新建仓库

在 github 新建一个名为{uisername}.github.io的仓库

将仓库与本地仓库进行关联

下载hexo-deployer-git插件之后，再目录下的_config.yml 文件中添加如下配置：

deploy:
  type: "git"
  repo: git@github.com:{username}/{username}.github.io
  branch: main

  1. 测试提交

hexo clean & hexo g & hexo d

  1. 选择一个主题

cd themes
git clone https://github.com/theme-particlex/hexo-theme-particlex.git particlex --depth=1

然后在根目录 _config.yml 设置主题为 ParticleX 即可。

theme: particlex

关闭闭自带代码高亮

Hexo 有自带的代码高亮，但是和 ParticleX 的不兼容。

highlight:
    enable: false
prismjs:
    enable: false

  1. 创建第一篇博客

Hexo 撰写博文也不难，分为 post 和 draft 两种，其中 post 存放在 source/_posts 目录下，draft 存放在 source/_drafts 目录下。

post 和 draft 的区别在于前者会被发布到博客，而后者不会被发布。

hexo n post "hexo-github 博客部署"

Hexo 会自动在 source/_posts 目录下新建一个名为 hexo-github-博客部署的 .md 文件；打开该文件，可以看到：

---
title: hexo-github 博客部署
date: 2023-10-02 21:28:30
tags:
---

tags 是该 post 的标签，可以添加自定义的标签：

tags: - demo - first-post

接着再进行本地调试三连，就可以看到刚刚写好的博文了。

hexo clean
hexo g
hexo s --debug
