grub rescue 修复grub开机引导

2024/1/29 linux

开机报错file '/bootgrub/xxxx/normal.mod' not found

这表示你的grub2的配置文件坏了……由于分区调整或分区UUID改变造成grub2不能正常启动，从而进入修复模式了（grub rescue)，也称救援模式。

在救援模式下只有很少的命令可以用：set , ls , insmod , root , prefix

set  查看环境变量，这里可以查看启动路径和分区。
ls   查看设备
insmod  加载模块
root  指定用于启动系统的分区,在救援模式下设置grub启动分区
prefix 设定grub启动路径
  • 首先ls展示出所有分区，然后ls 分区设备名称一个个进行查看，我查看到ls (hd0,gpt8)是查看到了文件信息
  • 修改启动分区
    • set root=(hd0,gpt8)
    • set prefix=(hd0,gpt8)/boot/grub
    • insmod normal
    • normal

执行完这几个命令即可正常启动

启动后，如果重启仍然会进入grub secure模式，因此需要修复，linux环境下执行

sudo update-grub
sudo grub-install /dev/nvme0n1 (这个路径应该是/boot挂载点对应的分区，`df-h`查看即可)
